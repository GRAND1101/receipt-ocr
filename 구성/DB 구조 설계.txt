DB 구조 설계 

1. users 테이블 (사용자 정보)
id (정수, PK): 사용자 고유 ID

email (텍스트): 사용자 이메일 (Google 로그인용, 중복 불가)

name (텍스트): 사용자 이름

created_at (날짜/시간): 가입 일시 (기본값: 현재 시간)

2. receipts 테이블 (영수증 정보)
id (정수, PK): 영수증 고유 ID

user_id (정수, FK): 사용자 ID (users 테이블 참조)

store_name (텍스트): 상호명 (OCR로 추출된 값)

total_amount (정수): 총 결제 금액

purchased_at (날짜/시간): 구매 일시

image_url (텍스트): 영수증 이미지 경로 또는 파일명

ocr_status (텍스트): OCR 처리 결과 상태 (예: success, fail)

3. receipt_items 테이블 (영수증 내 항목별 정보)
id (정수, PK): 항목 고유 ID

receipt_id (정수, FK): 연결된 영수증 ID (receipts 테이블 참조)

item_name (텍스트): 항목명 (예: 아메리카노, 삼각김밥)

category (텍스트): 소비 분류 (식비, 교통, 생활 등)

price (정수): 항목 금액

is_corrected (불리언): 사용자가 수동으로 수정했는지 여부

추가된 구조 예시 (텍스트 요약)
[선택] categories 테이블 (소비 항목 분류 정의)
id (정수, PK): 카테고리 ID

name (텍스트): 카테고리 이름 (식비, 교통 등)

color (텍스트): 차트 표시용 색상 HEX 코드

icon (텍스트): 카테고리용 아이콘 파일명 또는 코드

→ receipt_items.category는 categories.name과 연동 가능

[선택] logs 테이블 (사용자 활동 로그)
id (정수, PK)

user_id (정수, FK)

action_type (텍스트): 'upload', 'login', 'view_dashboard' 등

timestamp (날짜/시간): 발생 시각

extra_info (텍스트): 선택적 부가 데이터 (예: 파일명)
